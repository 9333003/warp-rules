# warp-rules

Утилита для серверов с Xray/VLESS: определяет, какие сервисы на сервере «сломаны» (определяются как Россия или заблокированы по региону), и выдаёт готовый блок маршрутизации для заворачивания их через WARP.

Построена как обёртка над [ipregion](https://github.com/vernette/ipregion).

## Что делает

1. Запускает `ipregion` и определяет **страну сервера по консенсусу** — берёт ответы всех GeoIP-сервисов и выбирает страну по большинству голосов (устойчиво к одиночным ошибкам отдельных баз).
2. Находит **сломанные сервисы** — те, что определяются как `RU` или имеют явный флаг недоступности (например `Gemini Supported: No`).
3. Проверяет **WARP-выход** — автоматически находит WireGuard/WARP-интерфейс и быстро определяет страну выхода через `cloudflare.com/cdn-cgi/trace` (без зависающего `ipregion -i`).
4. Выносит вердикт:
   - сервер не-RU, что-то сломано, WARP даёт нормальную страну → выдаёт готовый блок;
   - сервер RU, но WARP выходит в рабочую страну → всё равно чинит, выдаёт блок;
   - сервер RU и WARP тоже RU/отсутствует → сообщает «работа невозможна».
5. Печатает готовый JSON-блок доменов для секции `routing.rules`.

**YouTube не трогается никогда** — даже если определяется как RU. Это сделано намеренно: RU-выход для YouTube означает отсутствие рекламы, что является плюсом.

## Запуск

Одной командой на любом сервере (ничего ставить не надо — зависимости подтянутся):

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/9333003/warp-rules/main/warp-rules.sh)
```

Удобный алиас (выполнить один раз на каждом сервере):

```bash
echo "alias warp='bash <(curl -fsSL https://raw.githubusercontent.com/9333003/warp-rules/main/warp-rules.sh)'" >> ~/.bashrc
source ~/.bashrc
```

После этого достаточно набрать `warp`.

```

[!] Сломанные сервисы:
      - Google — определяется как RU
      - Gemini Supported — недоступен (No)
[*] Найден WARP-интерфейс: warp. Проверяю выход...

```

## Как применить результат

Скопируй выданный JSON-блок и вставь в секцию `routing.rules` своего Xray-конфига **выше** дефолтного маршрута (и выше любого широкого правила, ловящего весь Google). Не забудь запятую между правилами. Перезапусти Xray и проверь:
- сломанный сервис (Gemini) теперь работает;
- YouTube по-прежнему без рекламы (остался на прямом выходе).

## Требования

- `curl` — для запросов;
- `jq` — для разбора JSON (ставится автоматически при первом запуске `ipregion`);
- `wireguard-tools` (`wg`) — для проверки WARP-выхода. Если отсутствует, скрипт выдаст блок, но пометит его как «предварительный, проверь WARP сам».

## Настройка

В начале скрипта можно поменять:
- `WARP_TAG` — тег outbound, в который заворачивается трафик (по умолчанию `warp-out`);
- `BAD_COUNTRIES` — список «нерабочих» стран выхода (RU, CN и т.д.);
- `SKIP_SERVICES` — сервисы, которые никогда не трогать (по умолчанию YouTube);
- `SERVICE_DOMAINS` — таблица соответствия «сервис → домены/geosite».
